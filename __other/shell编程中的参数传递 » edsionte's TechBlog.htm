shell编程中的参数传递

2011年1月27日 由 edsionte 留言 »

在shell程序中引入参数，shell程序就会灵活很多。本文首先简单介绍参数传递中的一些基本知识点，再通过一个通讯录的举例来加深对shell编程中参数传递的理解。

1. 参数的引用

当运行shell程序时，shell会自动将命令行中第一个参数保存在变量1中，第二个参数保存在变量2中，依次类推。因此，当需要在shell程序中使用这些参数时，便可以通过$1和$2这样的方法来引用。比如：

view source
print?
1edsionte@edsionte-laptop:~/shelltest$ cat myargs
2echo "\"$1\" \"$2\" was passed"
3edsionte@edsionte-laptop:~/shelltest$ ./myargs edsionte wu
4"edsionte" "wu" was passed
5edsionte@edsionte-laptop:~/shelltest$ ./myargs edsionte
6"edsionte" "" was passed
7edsionte@edsionte-laptop:~/shelltest$ ./myargs edsionte wu me
8"edsionte" "wu" was passed

通过上述举例可以看到，命令行中的第n个参数会自动传递给相应的特殊变量n。通过$n就可以在shell程序中对其进行引用；当引用未赋值的特殊变量时，其结果将会显示空。

2. #变量

每当shell程序运行时，特殊变量#中就会保存此次shell程序中键入的参数个数。比如：

view source
print?
1echo "There are $# arguments passed"
2echo "\"$1\" \"$2\" was passed"
3edsionte@edsionte-laptop:~/shelltest$ ./myargs edsionte
4There are 1 arguments passed
5"edsionte" "" was passed

命令行中的参数是以空格作为间隔。因此，下面这种结果也很合理：

view source
print?
1edsionte@edsionte-laptop:~/shelltest$ ./myargs "edsionte wu"
2There are 1 arguments passed
3"edsionte wu" "" was passed
4edsionte@edsionte-laptop:~/shelltest$ name="edsionte wu"
5edsionte@edsionte-laptop:~/shelltest$ ./myargs $name
6There are 2 arguments passed
7"edsionte" "wu" was passed

3. *变量

特殊变量*中会保存shell程序运行时用户键入的所有参数。在参数不确定或参数数目易变时，该变量就很有用。

view source
print?
1edsionte@edsionte-laptop:~/shelltest$ cat myargs
2echo "There are $# arguments passed"
3echo "\"$*\" was passed"
4edsionte@edsionte-laptop:~/shelltest$ ./myargs edsionte wu is me
5There are 4 arguments passed
6"edsionte wu is me" was passed
7edsionte@edsionte-laptop:~/shelltest$ ./myargs edsionte wu
8There are 2 arguments passed
9"edsionte wu" was passed

4. 通信录举例

查找用户

现在需要通过用户姓名从通信录info中查找一个用户的个人信息。我们通过编写shell程序lu来实现这个功能。最简单的程序为：

view source
print?
01edsionte@edsionte-laptop:~/shelltest$ cat info
02edsionte wu:male:xian:18717375182
03sen wang:female:hanzhong:15678675545
04ting li:male:guangzhou:15784859345
05jedsionte:male:xian:13556567884
06edison:male:hongkong:1384742488
07yun zhang:female:beijing:18654737473
08edsionte@edsionte-laptop:~/shelltest$ cat lu
09# look someone up in the info book
10grep $1 info
11edsionte@edsionte-laptop:~/shelltest$ ./lu edsionte
12edsionte wu:male:xian:18717375182
13jedsionte:male:xian:13556567884

这个结果似乎并不令人满意，因为我们想找用户名为edsionte的用户。不过这个问题很好解决，我们可以这样改进程序：

view source
print?
1edsionte@edsionte-laptop:~/shelltest$ cat lu
2# look someone up in the info book
3grep ^$1 info
4edsionte@edsionte-laptop:~/shelltest$ ./lu edsionte
5edsionte wu:male:xian:18717375182

如果我们想通过全名edsionte wu来查找用户信息，就应该用双引号将全名作为一个整体引入其中。这样看似很完美，可是结果却不尽人意：

view source
print?
1edsionte@edsionte-laptop:~/shelltest$ ./lu "edsionte wu"
2grep: wu: 没有那个文件或目录
3info:edsionte wu:male:xian:18717375182

这是因为虽然edsionte wu作为一个参数传递给特殊变量1，但是shell将其替换后，grep命令并不认为edsionte wu是一个参数，而是将wu作为目标文件之一。为了避免这种情况，我们应该这样改进程序：

view source
print?
1edsionte@edsionte-laptop:~/shelltest$ cat lu
2# look someone up in the info book
3grep "^$1" info
4edsionte@edsionte-laptop:~/shelltest$ ./lu "edsionte wu"
5edsionte wu:male:xian:18717375182

这样就可以满足我们要求了。但是，如果info中的信息如下所示，那么上述lu程序还是不算完美。比如：

view source
print?
01edsionte@edsionte-laptop:~/shelltest$ cat info
02edsionte wu:male:xian:18717375182
03sen wang:female:hanzhong:15678675545
04edsionte wuee:male:xian:18717375182
05ting li:male:guangzhou:15784859345
06jedsionte:male:xian:13556567884
07edison:male:hongkong:1384742488
08yun zhang:female:beijing:18654737473
09edsionte@edsionte-laptop:~/shelltest$ ./lu "edsionte wu"
10edsionte wu:male:xian:18717375182
11edsionte wuee:male:xian:18717375182

因此，我们也应该在参数的尾部也做一些“手脚”，使得grep命令只匹配我们所传递的参数：

view source
print?
1edsionte@edsionte-laptop:~/shelltest$ cat lu
2# look someone up in the info book
3grep "^$1:" info
4edsionte@edsionte-laptop:~/shelltest$ ./lu "edsionte wu"
5edsionte wu:male:xian:18717375182

至此，查找程序完成。

添加用户

当向通讯录中增添新用户信息时，我们必须再编写一个add程序。增加用户程序很简单：

view source
print?
1edsionte@edsionte-laptop:~/shelltest$ cat add
2# add someone to the info book
3echo "$1:$2:$3:$4" >> info
4edsionte@edsionte-laptop:~/shelltest$ ./add elva female taibei 1335676567
5edsionte@edsionte-laptop:~/shelltest$ ./lu elva
6elva:female:taibei:1335676567

>>使得新的信息会追加到原文件的末尾。在上述命令的基础上，我们还可以再增加按姓名排序的功能：

view source
print?
01edsionte@edsionte-laptop:~/shelltest$ cat add
02# add someone to the info book
03echo "$1:$2:$3:$4" >> info
04sort -o info info
05edsionte@edsionte-laptop:~/shelltest$ cat info
06edison:male:hongkong:1384742488
07edsionte wuee:male:xian:18717375182
08edsionte wu:male:xian:18717375182
09elva:female:taibei:1335676567
10jedsionte:male:xian:13556567884
11sen wang:female:hanzhong:15678675545
12ting li:male:guangzhou:15784859345
13yun zhang:female:beijing:18654737473

添加程序结束。

删除用户

有了上述两个程序的基础，删除用户程序则变得容易很多。首先通过sed命令将完全匹配给定人名的用户信息从info中删除；因为sed命令并不改变info文件本身的内容，因此需要将sed的输出结果暂存至缓冲区；最后通过mv命令替换文件名即可。

view source
print?
01edsionte@edsionte-laptop:~/shelltest$ cat del
02# delete the given user from info
03sed "/^$1:/d" info > /tmp/info
04mv /tmp/info ./info
05edsionte@edsionte-laptop:~/shelltest$ ./del elva
06edsionte@edsionte-laptop:~/shelltest$ cat info
07edison:male:hongkong:1384742488
08edsionte wuee:male:xian:18717375182
09edsionte wu:male:xian:18717375182
10jedsionte:male:xian:13556567884
11sen wang:female:hanzhong:15678675545
12ting li:male:guangzhou:15784859345
13yun zhang:female:beijing:18654737473

随着对shell编程的深入理解，上述通讯录程序还可以不断完善。
